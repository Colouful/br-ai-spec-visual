# V1.1 Prisma / Seed 指南

## 数据库准备

`br-ai-spec-visual` 使用 Prisma(数据库 ORM) 保存运行态可视化数据，包括 Project State、Run Event、History、Incident。

执行数据库命令前请确认：

1. `.env` 或本地配置中的数据库连接可用。
2. 本地 `node_modules` 已存在。
3. Collector API(采集接口) 不接收源码、rawPrompt(原始提示词)、rawResponse(原始响应)、绝对路径。

## Prisma Validate

```bash
./node_modules/.bin/prisma validate --config prisma/prisma.config.ts
```

V1.1 验收结果：通过。

## Migration 状态检查

当前项目使用 Prisma config(Prisma 配置文件)：

```text
prisma/prisma.config.ts
```

建议在需要检查 migration 状态时执行：

```bash
npx prisma migrate status --config prisma/prisma.config.ts
```

本轮未新增 migration，不自动执行数据库变更。

## Prisma Client 生成

```bash
npm run prisma:generate
```

对应脚本：

```bash
prisma generate --config prisma/prisma.config.ts
```

## Seed 数据说明

已有 seed(种子数据)入口：

```bash
npm run prisma:seed
```

对应文件：

```text
prisma/seed.ts
```

Visual V1.1 smoke test 不强依赖预置业务数据，但页面访问和工作区路由需要已有 workspace / project 数据。

## 常见失败原因

1. Prisma Client 未生成：执行 `npm run prisma:generate`。
2. 数据库连接不可用：检查 `.env`。
3. Collector payload 被拒绝：检查是否包含 sourceCode、fileContent、rawPrompt、rawResponse、绝对路径或密钥字段。
4. 页面跳转登录页：说明页面服务可访问，但当前浏览器会话未登录。

## 回滚建议

1. Collector 写入的是运行态结构化数据，可按 projectId / runId 定位清理。
2. 不要直接清空生产运行态表。
3. 如 seed 写入错误，优先用幂等 eventId 修正或手工删除测试 projectId。

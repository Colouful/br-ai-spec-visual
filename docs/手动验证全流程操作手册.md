# 手动验证全流程操作手册（核心 + 管理面）

> **目标**：一份可以让任何人坐到本机前，照着勾完就能确认 visual + auto 全部主要功能可用的手册。
> **执行环境**：本地（macOS）。visual 跑在 `http://localhost:18780`，DB 在 docker `br-ai-spec-visual-db-1`，auto 用本地源码 `node $AUTO_PATH/bin/cli.js`。
> **配套文档**：
> - [docs/端到端安装测试-init-update-sync.md](./端到端安装测试-init-update-sync.md)（自动化场景 + 已知问题）
> - [docs/测试文件.md](./测试文件.md)（visual 桥接合规性专项）

每个模块结构：**📖 背景** → **🎬 操作步骤** → **☑️ 勾选清单**。
所有勾选项都直接对应当前仓库里真实存在的页面/接口。

## 1. 登录与角色权限

### 📖 背景
visual 用 cookie session，admin 才看得到 `/admin/*` 与 `/platform/*`。

### 🎬 操作步骤

1. 打开 `http://localhost:18780/` → 应被重定向到 `/login`
2. 用 `admin` / `Console#2026` 登录，停在 `/overview`
3. 右上角下拉登出 → 切到 `viewer` / `Console#2026` 登录
4. 直接访问 `/admin/installations` → 应被拒/跳走（提示"无权限"）
5. 切回 `admin`

### ☑️ 勾选清单

- [ ] 未登录访问 `/overview` 被拒
- [ ] 三种角色（admin / maintainer / viewer）都能登录成功
- [ ] viewer 看不到平台治理菜单（`/platform/installations` / `/platform/members` / `/platform/settings`）
- [ ] viewer 看不到工作区"设置"项（`/w/[slug]/settings` 需要 admin）
- [ ] maintainer 能看到 `runs` / `changes`（viewer 看不到）
- [ ] 登出后所有受保护页面都跳到 `/login?next=...`

## 5. auto CLI：init / update / sync / check / uninstall

### 📖 背景

### 5.1 init

```bash
cd "$PROJ"

```

- [ ] 末尾打印 `规范与配置文件已同步到项目`

### 5.2 配置 visual-bridge 指向本地

```bash
node -e "
const fs=require('fs');
const f='.ai-spec/visual-bridge.json';

j.server_url=process.env.VISUAL_URL;
j.workspace_id=process.env.WS_ID;
fs.writeFileSync(f, JSON.stringify(j,null,2)+'\n');
console.log(j);
"
```

- [ ] 输出里 `server_url` / `workspace_id` 已是真实值

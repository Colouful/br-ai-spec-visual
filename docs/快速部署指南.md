# br-ai-spec-visual 快速部署指南

> 本文档说明如何在内网服务器快速部署 br-ai-spec-visual 可视化监控平台。

## 前置条件

### 硬件要求

- **CPU**: 2 核及以上
- **内存**: 4GB 及以上
- **磁盘**: 20GB 及以上（建议 SSD）

### 软件要求

- **Docker**: 20.10 或更高版本
- **Docker Compose**: 2.0 或更高版本
- **Git**: 任意版本
- **网络**: 内网可访问，端口 3000 可开放

### 前置准备

- 已有至少 1 个接入 ai-spec-auto 的项目
- 获取 visual 仓库访问权限

## 一键部署步骤（5 步）

### 步骤 1：克隆仓库并配置环境

```bash
# 1.1 克隆仓库
git clone http://git.internal/br-ai-spec-visual.git
cd br-ai-spec-visual

# 1.2 复制环境变量模板
cp .env.example .env

# 1.3 编辑 .env 文件
vim .env
# 设置以下变量：
# DB_PASSWORD=your_secure_password_here
# NEXTAUTH_SECRET=$(openssl rand -base64 32)
# NEXTAUTH_URL=http://your-server-ip:3000
```

### 步骤 2：启动服务

```bash
# 2.1 启动服务（后台运行）
docker-compose up -d

# 2.2 查看启动日志
docker-compose logs -f visual

# 2.3 等待服务启动（约 30-60 秒）
# 看到 "Ready on http://0.0.0.0:3000" 表示成功
```

### 步骤 3：验证服务状态

```bash
# 3.1 检查容器状态
docker-compose ps
# 预期：visual 和 db 都是 "Up" 状态

# 3.2 健康检查
curl http://localhost:3000/api/health
# 预期返回：{"status":"ok"}

# 3.3 执行数据库迁移（首次部署需要）
docker-compose exec visual npx prisma migrate deploy
```

### 步骤 4：配置目标项目

```bash
# 4.1 进入目标项目目录
cd /path/to/your/project

# 4.2 创建 visual 配置文件
cat > .ai-spec/visual-config.json <<EOF
{
  "enabled": true,
  "visual_url": "http://your-server-ip:3000",
  "workspace_id": "your-project-name",
  "workspace_name": "XX业务前端项目"
}
EOF

# 4.3 提交配置文件（可选）
git add .ai-spec/visual-config.json
git commit -m "feat: 接入 visual 监控"
```

### 步骤 5：批量采集历史数据

```bash
# 5.1 执行 Collector 命令
npx br-ai-spec-visual-collector \
  --workspace-id your-project-name \
  --project . \
  --server http://your-server-ip:3000

# 5.2 查看采集结果
# 预期输出：
# {
#   "workspace_id": "your-project-name",
#   "baseline": { "runs_found": 15, ... },
#   "raw_event_count": 120,
#   "ingest": { "inserted": 120, "skipped": 0 }
# }

# 5.3 在浏览器访问控制台
# http://your-server-ip:3000/workspaces
# 应该能看到工作区和历史 run 记录
```

## 验证清单（5 项）

完成部署后，请逐项验证：

- [ ] **服务健康**：`curl http://server-ip:3000/api/health` 返回 OK
- [ ] **控制台访问**：浏览器打开 `http://server-ip:3000/workspaces` 正常显示
- [ ] **历史数据**：控制台能看到工作区和历史 run 记录
- [ ] **实时推送**：执行 `/spec-start` 后，控制台实时显示新 run
- [ ] **降级验证**：停止 visual 服务，auto 正常运行

## 常见问题排查

### 问题 1：容器启动失败

**症状**：

```
Error response from daemon: Conflict. The container name "/br-ai-spec-visual" is already in use
```

**解决方案**：

```bash
# 停止并删除旧容器
docker-compose down

# 重新启动
docker-compose up -d
```

### 问题 2：数据库连接失败

**症状**：

```
Error: P1001: Can't reach database server at `db:3306`
```

**解决方案**：

```bash
# 1. 检查数据库容器状态
docker-compose ps db

# 2. 查看数据库日志
docker-compose logs db

# 3. 等待数据库初始化完成（首次启动需要 30 秒）
# 4. 重启 visual 服务
docker-compose restart visual
```

### 问题 3：端口被占用

**症状**：

```
Error starting userland proxy: listen tcp4 0.0.0.0:3000: bind: address already in use
```

**解决方案**：

```bash
# 方案 1：修改 .env 文件中的 PORT
echo "PORT=3001" >> .env
docker-compose up -d

# 方案 2：停止占用端口的进程
lsof -ti:3000 | xargs kill -9
docker-compose up -d
```

### 问题 4：Collector 推送失败

**症状**：

```
Error: connect ECONNREFUSED 10.0.1.100:3000
```

**解决方案**：

```bash
# 1. 检查 visual 服务是否启动
docker-compose ps

# 2. 检查防火墙是否开放端口 3000
sudo firewall-cmd --list-ports

# 3. 尝试从目标机器访问 visual
curl http://10.0.1.100:3000/api/health

# 4. 如需开放端口
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --reload
```

### 问题 5：WebSocket 连接失败

**症状**：

浏览器控制台报错：`WebSocket connection failed`

**解决方案**：

如果使用 Nginx 反向代理，需要配置 WebSocket 支持：

```nginx
location / {
  proxy_pass http://localhost:3000;
  proxy_http_version 1.1;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection "upgrade";
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

## 管理命令

### 查看日志

```bash
# 查看所有容器日志
docker-compose logs -f

# 只查看 visual 日志
docker-compose logs -f visual

# 只查看 db 日志
docker-compose logs -f db

# 查看最近 100 行日志
docker-compose logs --tail=100 visual
```

### 重启服务

```bash
# 重启所有服务
docker-compose restart

# 只重启 visual
docker-compose restart visual

# 只重启 db
docker-compose restart db
```

### 停止服务

```bash
# 停止服务但保留数据
docker-compose stop

# 停止并删除容器（数据保留）
docker-compose down

# 停止并删除容器和数据卷（⚠️ 数据丢失）
docker-compose down -v
```

### 更新部署

```bash
# 1. 拉取最新代码
git pull

# 2. 重新构建镜像
docker-compose build

# 3. 重启服务
docker-compose up -d

# 4. 执行数据库迁移（如有新迁移）
docker-compose exec visual npx prisma migrate deploy
```

### 数据库备份

```bash
# 备份数据库
docker-compose exec db mysqldump -uroot -p${DB_PASSWORD} ${DB_NAME} > backup.sql

# 恢复数据库
cat backup.sql | docker-compose exec -T db mysql -uroot -p${DB_PASSWORD} ${DB_NAME}
```

## 性能优化建议

### 1. 使用 SSD 存储

数据库数据卷建议挂载到 SSD 上，提高查询性能：

```yaml
volumes:
  db_data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /mnt/ssd/visual-db-data
```

### 2. 调整数据库配置

创建 `my.cnf` 文件并挂载到数据库容器：

```ini
[mysqld]
innodb_buffer_pool_size=1G
max_connections=200
```

修改 `docker-compose.yml`：

```yaml
db:
  volumes:
    - ./my.cnf:/etc/mysql/conf.d/custom.cnf
```

### 3. 启用日志轮转

创建 `logrotate.conf`：

```conf
/app/logs/*.log {
  daily
  rotate 7
  compress
  missingok
  notifempty
}
```

### 4. 配置反向代理

使用 Nginx 作为反向代理，启用缓存和压缩：

```nginx
http {
  upstream visual {
    server localhost:3000;
  }

  server {
    listen 80;
    server_name visual.internal.company.com;

    gzip on;
    gzip_types text/plain application/json;

    location / {
      proxy_pass http://visual;
      proxy_cache visual_cache;
      proxy_cache_valid 200 5m;
    }
  }
}
```

## 安全建议

### 1. 修改默认密码

⚠️ **务必修改 `.env` 中的 `DB_PASSWORD` 和 `NEXTAUTH_SECRET`**：

```bash
# 生成安全的密码
openssl rand -base64 32
```

### 2. 限制数据库端口访问

如果不需要外部访问数据库，删除 `docker-compose.yml` 中的 `ports` 映射：

```yaml
db:
  # ports:
  #   - "3306:3306"  # 删除或注释此行
```

### 3. 使用 HTTPS

配置 Nginx + Let's Encrypt 证书：

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d visual.internal.company.com
```

### 4. 配置防火墙

```bash
# 只允许内网访问
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port protocol="tcp" port="3000" accept'
sudo firewall-cmd --reload
```

## 监控和告警

### 健康检查脚本

创建 `health-check.sh`：

```bash
#!/bin/bash
VISUAL_URL=${1:-http://localhost:3000}
HTTP_STATUS=$(curl -s -o /dev/null -w "%{http_code}" "${VISUAL_URL}/api/health")

if [ "$HTTP_STATUS" -ne 200 ]; then
  echo "❌ Visual service unhealthy: ${HTTP_STATUS}"
  exit 1
fi

echo "✅ Visual service healthy"
```

### 添加到 Cron

```bash
# 每 5 分钟检查一次
*/5 * * * * /path/to/health-check.sh || mail -s "Visual Down" admin@company.com
```

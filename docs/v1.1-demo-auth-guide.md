# V1.1 Demo Auth Mode 使用指南

## 1. 用途

Demo Auth Mode(演示登录模式) 用于本地验证、V1.1 演示和 V2 试点演示时临时跳过页面登录，让演示人员可以直接访问 Visual 页面。

它只影响页面鉴权，不影响 Collector API(采集接口)。

## 2. 适用场景

- 本地开发验证。
- V1.1 演示。
- V2 试点演示。
- 内网临时演示环境。

## 3. 不适用场景

- 生产环境。
- 正式权限验证。
- 对外公开环境。
- 需要验证真实用户登录、角色、session(会话) 或 token(令牌) 的场景。

## 4. 环境变量配置

```env
VISUAL_AUTH_MODE=demo
VISUAL_DEMO_USER_ID=demo-user
VISUAL_DEMO_USER_NAME=演示用户
VISUAL_DEMO_TEAM_ID=demo-team
```

字段说明：

- `VISUAL_AUTH_MODE`：设置为 `demo` 时开启演示登录模式。
- `VISUAL_DEMO_USER_ID`：演示用户 ID，默认 `demo-user`。
- `VISUAL_DEMO_USER_NAME`：演示用户名称，默认 `演示用户`。
- `VISUAL_DEMO_TEAM_ID`：演示团队 ID，默认 `demo-team`。

Demo 用户结构：

```ts
{
  id: "demo-user",
  name: "演示用户",
  role: "admin",
  teamId: "demo-team",
  isDemo: true
}
```

## 5. 如何开启

本地启动示例：

```bash
VISUAL_AUTH_MODE=demo \
VISUAL_DEMO_USER_ID=demo-user \
VISUAL_DEMO_USER_NAME=演示用户 \
VISUAL_DEMO_TEAM_ID=demo-team \
npm run dev
```

访问页面：

```text
http://172.20.10.4:18780/w/test_%E5%89%AF%E6%9C%AC17
```

页面顶部会显示：

```text
当前处于演示模式，仅用于本地验证和试点演示。
```

## 6. 如何关闭

关闭方式：

```bash
unset VISUAL_AUTH_MODE
unset VISUAL_DEMO_USER_ID
unset VISUAL_DEMO_USER_NAME
unset VISUAL_DEMO_TEAM_ID
npm run dev
```

或者显式设置非 demo 值：

```bash
VISUAL_AUTH_MODE=off npm run dev
```

关闭后页面继续走原有登录流程。

## 7. 生产环境禁止说明

生产环境禁止开启演示登录模式。

当满足以下条件时，服务会直接抛出中文错误：

```text
NODE_ENV=production
VISUAL_AUTH_MODE=demo
```

错误信息：

```text
生产环境禁止开启演示登录模式
```

该保护不能通过其它配置绕过。

## 8. 和 Collector API 的关系

Visual Collector API 不依赖页面登录态，Demo Auth 只影响页面访问。

以下接口保持原有逻辑：

- `POST /api/collector/project-state`
- `POST /api/collector/run-event`
- `POST /api/collector/history`
- `POST /api/collector/incident`

Collector API 的隐私校验不会因为 Demo Auth Mode 而降低。

## 9. 为什么不读取数据库 token

Demo Auth Mode 是演示专用的本地页面鉴权分支，不代表真实用户登录。

因此它不会：

- 查询数据库 session(会话)。
- 读取数据库 token(令牌)。
- 创建用户记录。
- 写入 demo 用户到数据库。

这样可以避免把演示用户误认为线上真实用户。

## 10. 为什么不伪造线上 token

Demo Auth Mode 不生成真实 token，也不伪造线上 token。

原因：

- 避免绕过正式认证链路。
- 避免污染真实 session(会话)。
- 避免演示配置被误用于生产环境。
- 保证生产环境仍由原有登录流程负责。

## 11. 常见问题

### Q：开启 demo 后 Collector API 会自动登录吗？

不会。Collector API 是服务端采集入口，不依赖页面登录态。

### Q：生产环境能不能临时开启 demo？

不能。`NODE_ENV=production` 且 `VISUAL_AUTH_MODE=demo` 会被阻断。

### Q：demo 用户是不是数据库用户？

不是。demo 用户只存在于服务端运行时返回值中，不写入数据库。

### Q：demo 用户是什么权限？

固定为 `admin(管理员)`，仅用于本地验证和演示。

### Q：关闭 demo 后会恢复原登录页吗？

会。关闭 `VISUAL_AUTH_MODE=demo` 后继续走现有登录逻辑，不会隐藏或删除登录功能。
